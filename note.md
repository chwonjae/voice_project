# 요청 streamlit 소프트웨어 한줄 설명
Streamlit로 구현하지만, 실제 모바일 은행 앱처럼 보이는 세로형 데모 앱. **“신한 SOL Bank 느낌의 모바일 뱅킹 시연 프로토타입”**입니다
## GOAL
사용자가 처음 봤을 때 **“진짜 은행 앱 같다”**고 느끼게 만들고, 그 안에서 계좌 선택 → 받는 사람 입력 → 금액 입력까지의 송금 플로우가 매우 자연스럽게 이어지도록 만드는 것
차후 FAST API와의 연동 (http 통신)을 고려하고 해당 기능들이 sample 기능이라는 것을 인지 할 수 있도록 변수, 함수 명들을 확실하게 작성할 것(차후 개발자가 제거하며 구현할 수 있도록)
## 주요 플로우
1. 진입 시 splace 화면 (💶 SUL Bank)
2. 메인 화면 (대부분이 기능을 하지 않는 디자인용(목업)으로 설계), 회색 아주 연한 그레이 바탕화면
- 최상위 sul bank(왼쪽 좌편) 로고, 오른쪽 (검색 버튼과 알림(종모양) 버튼)
- 그 하단은 은행 > 탭이다
  * [은행 탭 버튼]탭 타이틀 밑에 블루 라운드 상자로 (SUL 마크 💶 SUL 주거래 우대통장\n가상 계좌번호\n가상 현재 금액 오른쪽에 이체 마크가 있음). 해당 버튼은 유일하게 동작하는 실제 버튼이다, 해당 버튼을 클릭하면 이체 페이지로 이동할 예정
- 그 하단은 카드 > 탭이다.
  * 탭 타이틀 밑에 화이트 라운드 상자로 가상 카드에 대한 정보가 있다.
- 그 하단 부터는 창작하여 어울리는 탭을 추가한다(실제 기능은 하지 않는다. 증권/보험 등 실제 모바일 앱에 있을 듯한 탭 가상)
- 최하단에는 네이게이션 하단 탭이 존재하다 (홈 버튼이 활성화 되어 있고 금융, 혜택, 주식, 전체메뉴 햄버거 버튼 등이 있다.)/
3. 은행 탭 버튼 클릭시 이동되는 페이지, 송금 페이지, 누구에게 보낼까요?
- 최상단에는(타이틀) 뒤로가기 버튼(<), 받는분 선택 타이틀 텍스트, 오른쪽 상단에는 홈 버튼(집 모양 버튼)
- 밑에는 누구에게 보낼까요? (나눔 고딕 bold)
- 밑에는 기관 선택 select box(클릭시 여러 은행을 선택할 수 있는 popup box가 뜬다.)
- 밑에는 계좌번호 입력 edit box가 있다.
- 최하단에는 확인 버튼이 있다(보낼 금액 페이지로 이동)
4. 보낼 금액 입력 페이지
- 최상단에는(타이틀) 뒤로가기 버튼(<), 보낼 금액 입력 타이틀 텍스트, 오른쪽 상단에는 홈 버튼(집 모양 버튼)
- 밑에는 얼마를 보낼까요?
- 최하단에는 가상 키보드(숫자) 표시 (키보드 위에는 파란색 전송 버튼이 있음, width는 키보드와 같은 디자인)
